# 数据模型与事件协议

## 1. 设计原则

本项目的模型和事件协议用于约束 AI 辅助开发，防止字段命名、状态含义和模块通信在实现过程中漂移。

原则：

- TypeScript 类型是开发期约束。
- IPC、插件、文件、外部 Agent 输入必须做运行时校验。
- 内部 TypeScript 使用 `camelCase`。
- 外部 JSON 协议优先使用 `snake_case`。
- Event Bus 只转发事件，不做业务判断。
- 各模块拥有自己的状态。

## 2. 命名规则

内部代码：

```ts
sessionId
projectPath
createdAt
```

外部 JSON：

```json
{
  "session_id": "xxx",
  "project_path": "D:/demo",
  "created_at": "2026-05-31T10:00:00+08:00"
}
```

时间格式：

```text
ISO 8601 string
示例：2026-05-31T10:00:00+08:00
```

ID 规则：

```text
todo_xxx
reminder_xxx
agent_session_xxx
agent_event_xxx
memory_xxx
share_xxx
```

## 3. 核心类型

### 3.1 PetState

```ts
export type PetState =
  | 'idle'
  | 'working'
  | 'reminding'
  | 'success'
  | 'error'
  | 'waiting'
  | 'sleeping'
  | 'sharing';
```

含义：

- `idle`：待机。
- `working`：正在帮用户盯任务或处理事情。
- `reminding`：普通提醒。
- `success`：任务完成或庆祝。
- `error`：错误、失败或异常。
- `waiting`：等待用户确认。
- `sleeping`：勿扰、休息或低活跃状态。
- `sharing`：生成分享内容。

## 5. IPC 通道

IPC 通道集中放在：

```text

```

建议定义：

```ts
export const IpcChannels = {
  ChatSendMessage: 'chat:send-message',
  TodoCreate: 'todo:create',
  TodoList: 'todo:list',
  TodoComplete: 'todo:complete',
  TodoDelete: 'todo:delete',
  ReminderList: 'reminder:list',
  ReminderSnooze: 'reminder:snooze',
  AgentGetStatus: 'agent:get-status',
  AgentGetHistory: 'agent:get-history',
  MemoryList: 'memory:list',
  MemoryCreate: 'memory:create',
  MemoryDelete: 'memory:delete',
  SharePreview: 'share:preview',
  ShareGenerate: 'share:generate',
  SettingsGetPetConfig: 'settings:get-pet-config',
  SettingsUpdatePetConfig: 'settings:update-pet-config',
  SettingsGetNotificationConfig: 'settings:get-notification-config',
  SettingsUpdateNotificationConfig: 'settings:update-notification-config'
} as const;
```

不要提供无约束的：

```text
settings:get(anyKey)
settings:set(anyKey, anyValue)
```

如果必须提供 key/value API，Main Process 必须校验 key 白名单。

## 6. 插件 Manifest

```ts
export interface PluginManifest {
  id: string;
  name: string;
  version: string;
  description?: string;
  permissions: PluginPermission[];
  events: string[];
}

export type PluginPermission =
  | 'process:read'
  | 'window:read'
  | 'notification:send'
  | 'pet:react'
  | 'memory:suggest'
  | 'network:deepseek'
  | 'storage:read'
  | 'storage:write';
```

MVP 阶段插件权限主要用于文档化和设置页展示。P1 开始由 Plugin Runtime 在调用系统能力前强制校验 permissions。

## 8. 运行时校验

TypeScript 类型不能替代运行时校验。

必须校验的数据来源：

- Renderer 到 Main 的 IPC 输入。
- DeepSeek 返回内容。
- Agent 监听来源。
- 插件 manifest。
- 本地存储读取结果。
- 导入文件。

MVP 可以先手写校验函数，P1 再考虑 Zod。

## 9. 协议变更规则

变更任何核心协议时，必须同步修改：

- `数据模型与事件协议.md`
- `src/shared/types.ts`
- `src/shared/eventNames.ts`
- `src/shared/ipcChannels.ts`
- 对应 smoke test

禁止只改实现不改协议文档。
