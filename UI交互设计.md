# UI交互设计

## 1. 设计目标

桌宠 UI 的核心不是“功能面板”，而是一个常驻桌面的轻量陪伴层。

目标：

- 可爱但不幼稚。
- 主动但不骚扰。
- 常驻但不挡操作。
- 信息短、准、可行动。
- 适合 AI先锋使用者分享。

## 2. 主界面

主界面就是桌宠本体，右侧附带 Agent 状态指示灯。

组成：

- 桌宠 Canvas。
- Agent 状态指示灯（多 Agent 同步显示）。
- 聊天气泡。
- 提醒气泡。
- 右键菜单。

不要做传统首页。启动后直接出现桌宠。

### 2.1 Agent 状态指示灯

指示灯绝对定位在桌宠右侧，和桌宠同窗口：

- 每个已接入 Agent 一行：颜色圆点 + Agent 名 + 状态文字 + 运行时长。
- 面板收起时简化显示（只显示圆点+名称）。
- 无接入 Agent 时不显示。
- 和桌宠一起拖拽/隐藏。

## 3. 桌宠状态

P0 状态：

```text
idle
working
reminding
success
error
waiting
sleeping
sharing
```

状态表现：

- `idle`：眨眼、轻微晃动。
- `working`：看屏幕、思考、敲键盘。
- `reminding`：举牌、挥手。
- `success`：跳跃、庆祝。
- `error`：皱眉、低落。
- `waiting`：盯着用户、举手。
- `sleeping`：坐下、打瞌睡（勿扰模式或长时间无操作触发）。
- `sharing`：摆 pose、生成截图（Agent 完成触发分享入口时）。

## 4. 主动互动等级

```text
L0 纯视觉：只动，不打扰。
L1 轻气泡：短句，不发声。
L2 标准提醒：气泡 + 可选系统通知。
L3 强提醒：权限确认等高优先级状态，持续提示直到响应。
```

规则：

- 默认不频繁使用 L1。
- 待办到期使用 L2。
- Agent 等待确认使用 L3。
- P0：提供手动勿扰模式和一键隐藏气泡。
- P1：检测到全屏应用时降级为 L0。
- P2：识别会议/录屏软件和屏幕共享场景，隐藏气泡和提醒内容。

## 5. 聊天气泡

触发：

- 点击桌宠。
- 快捷键。
- 右键菜单。

规则：

- 气泡跟随桌宠位置。
- 自动避开屏幕边界。
- 文案短，不像客服。
- 支持复制回复。
- DeepSeek 离线时显示降级提示。

输入：

- 单行输入优先。
- 支持 Enter 发送。
- 支持 Shift+Enter 换行。

## 6. 待办提醒

提醒气泡包含：

- 标题。
- 时间。
- 操作按钮：完成、稍后、忽略。

规则：

- 同一提醒不重复刷屏。
- 用户正在全屏时降级（P1）。
- 稍后提醒默认 10 分钟，可后续配置。
- 过期提醒在应用启动时汇总展示，不逐条轰炸。

## 7. Agent 提醒

状态文案示例：

```text
started：我帮你盯着它。
running：它还在跑。
waiting_permission：它在等你确认。
completed：跑完了，要看看结果吗？
failed：这里失败了，我可以帮你总结。
idle_too_long：它好像卡住一会儿了。
```

规则：

- `waiting_permission` 使用 L3。
- `completed` 使用 L2，并出现分享入口。
- `failed` 使用 L2，不自动分享。
- 低置信度状态只提示“可能”。

## 8. 右键菜单

P0 菜单：

```text
聊天
新增待办
帮我盯这个任务
生成分享图
勿扰模式
设置
隐藏
退出
```

规则：

- 菜单项少而稳定。
- 不把所有功能都塞进去。
- 调试项仅开发模式显示。

## 9. 设置页

P0 设置：

- DeepSeek API Key。
- 桌宠名字。
- 用户称呼。
- 通知开关。
- 勿扰时间。
- 分享水印。
- 清空本地数据。

P1 设置：

- 记忆管理。
- 插件管理。
- 动作包。
- 分享模板。
- Agent Watcher 配置。

## 10. 分享预览

分享页流程：

```text
生成内容
  -> 脱敏
  -> 预览
  -> 用户确认
  -> 保存 PNG
```

规则：

- 不自动保存。
- 不自动上传。
- 默认隐藏敏感路径。
- 用户可以编辑文案。
- 保存和复制按钮并列。

## 11. 鼠标穿透

透明窗口要避免挡住用户。

规则：

- 非交互区域尽量穿透。
- 桌宠本体可点击。
- 气泡区域可点击。
- 拖拽时关闭穿透。
- 拖拽结束恢复穿透。

MVP 如果不能做精细穿透，必须控制窗口尺寸，避免透明区域覆盖大面积桌面。

## 12. 空状态

不要空白页。

示例：

- 没有待办：“现在还没有任务，要不要让我提醒你一件事？”
- 没有记忆：“你可以说‘记住，我喜欢简短提醒’。”
- 没有 Agent：“可以先用模拟器看看我怎么盯任务。”

## 13. 错误状态

技术错误要转成人话：

- 网络断开：“网络好像不太对。”
- DeepSeek 超时：“它想得有点久。”
- API Key 未配置：“先把 DeepSeek Key 填上，我才能聊天。”
- 存储失败：“这条我没记住，可能本地存储出问题了。”

## 14. P0 验收

- 桌宠启动后即为主界面。
- 点击可聊天。
- 待办提醒可操作。
- Agent 状态可触发动画。
- 分享图必须预览。
- P0 手动勿扰和一键隐藏气泡可用。
- 透明窗口不大面积挡鼠标。
