# MVP范围与验收标准

## 1. MVP 目标

第一版目标不是做完整平台，而是验证一个闭环：

```text
AI先锋使用者是否愿意让一个桌面 AI 宠物常驻，
并用它聊天、管理提醒、盯 AI Agent 状态、生成可分享内容。
```

MVP 要回答四个问题：

1. 桌宠常驻桌面是否足够轻、稳、不烦人。
2. DeepSeek 聊天和待办提醒是否能形成日常使用价值。
3. Agent 状态提醒是否有明显差异化。
4. 分享图是否让用户愿意传播。

## 2. P0 必须完成

### 2.1 桌宠窗口

功能：

- Windows 透明无边框窗口。
- 默认置顶。
- 显示默认桌宠形象。
- 支持拖拽移动。
- 支持右键菜单。
- 支持托盘退出。

验收：

- 应用可在 Windows 启动。
- 桌宠窗口透明背景正常。
- 拖拽无明显卡顿。
- 右键菜单可打开设置、隐藏、退出。
- 退出后进程不残留。

### 2.2 Canvas 桌宠动画

功能：

- 加载默认 sprite sheet。
- 支持 `idle`、`working`、`reminding`、`success`、`error` 五种状态。
- 待机动画 8-12 FPS。
- 响应动画 24 FPS。

验收：

- React 不逐帧重渲染。
- 切换状态无明显卡顿。
- 隐藏窗口或勿扰模式下降帧或暂停。
- 空闲 CPU 尽量低于 2%。

### 2.3 DeepSeek 聊天

功能：

- 用户配置 DeepSeek API Key。
- 点击桌宠弹出聊天气泡。
- 用户发送文本，桌宠返回回复。
- DeepSeek 不可用时显示降级提示。

验收：

- API Key 不暴露到 renderer 日志。
- DeepSeek 请求由 Main Process 发起。
- 网络失败不影响桌宠、待办、提醒。
- 聊天气泡不遮挡桌宠主要交互。

### 2.4 待办与提醒

功能：

- 创建待办。
- 通过 DeepSeek 从简单自然语言中创建待办。
- 完成待办。
- 删除待办。
- 设置提醒时间。
- 到点触发桌宠气泡和系统通知。
- 支持稍后提醒、完成、忽略。

验收：

- 应用隐藏后提醒仍能触发。
- 重复触发不会刷屏。
- DeepSeek 离线时仍能手动创建待办。
- DeepSeek 在线时，能解析简单句式，例如“10 分钟后提醒我喝水”。
- 应用启动时能恢复未完成待办。

### 2.5 Agent 状态模拟器

功能：

- 开发调试面板手动触发 Agent 状态。
- 支持 `started`、`running`、`waiting_permission`、`completed`、`failed`。
- 状态进入标准 `AgentEvent`。
- Pet Engine 根据事件切换表现。

验收：

- 状态变化 500ms 内反映到桌宠。
- `waiting_permission` 优先级高于普通提醒。
- `completed` 可触发分享入口。
- 所有事件可在调试日志中查看。

### 2.6 分享图

功能：

- Agent 完成后出现分享入口。
- 生成静态分享图。
- 分享前预览。
- 基础隐私过滤。
- 保存 PNG。

验收：

- 分享图可读、尺寸稳定。
- 生成前不会自动保存。
- 默认脱敏 API Key、Token、绝对路径、邮箱、手机号。
- 不生成 GIF/MP4。

## 3. P0.5 / P1 延后完成

- 基础记忆系统：`记住 X`、`忘记 X`、本地最多 20 条记忆。
- 记忆管理页。
- P0.5 / 内测增强：真实 Agent Watcher 第一种工具集成。
- P1：真实 Agent Watcher 稳定化和第二个工具集成。
- 今日总结卡片。
- 分享图模板选择。
- 系统托盘更多状态。
- SQLite 迁移。
- 插件启用/禁用。

## 4. P2 暂不考虑

- Live2D。
- 3D VRM。
- 多模型供应商。
- 云同步。
- 插件市场。
- 创作者市场。
- 团队版。
- 自动更新。
- 代码签名。
- GIF/MP4 导出。

## 5. 性能验收

MVP 目标：

```text
空闲 CPU：尽量 < 2%
常驻内存：尽量 < 250MB
待机动画：8-12 FPS
响应动画：24 FPS
Agent 模拟事件响应：< 500ms
普通提醒触发延迟：< 1s
```

说明：

- 内存目标以应用总进程组为准。
- 开发模式可放宽，生产包需记录实际数据。
- 如果空闲 CPU 超过目标，优先检查动画重绘和后台轮询。

## 6. 安全验收

MVP 必须满足：

- `contextIsolation: true`
- `nodeIntegration: false`
- `sandbox: true`
- preload 只暴露白名单 API。
- 不暴露完整 `ipcRenderer`。
- DeepSeek API Key 不进入 renderer。
- IPC 输入做运行时校验。
- 分享前预览和脱敏。
- 不加载远程页面作为主 UI。

## 7. 演示验收路径

MVP 完成时必须能录制以下演示：

1. 启动应用，桌宠出现在桌面。
2. 拖拽桌宠到屏幕角落。
3. 点击桌宠，输入一句话，DeepSeek 回复。
4. 创建一个 10 秒后的提醒。
5. 提醒到点，桌宠弹气泡并切换提醒状态。
6. 打开 Agent 模拟器，触发 `waiting_permission`。
7. 桌宠切换强提醒状态。
8. 触发 `completed`。
9. 桌宠庆祝并生成分享图。
10. 用户预览分享图后保存。

## 8. MVP 成功标准

内部版本成功标准：

- 连续运行 2 小时不崩溃。
- 空闲状态没有明显风扇和卡顿。
- 所有 P0 功能可演示。
- smoke test 能跑通。
- 至少 3 个非开发者用户能理解产品价值。

公开测试成功标准：

- 30% 测试用户完成 DeepSeek 配置和一次聊天。
- 30% 测试用户创建至少一个待办提醒。
- 20% 测试用户触发 Agent 状态提醒或看懂演示价值。
- 15% 测试用户生成分享图。
