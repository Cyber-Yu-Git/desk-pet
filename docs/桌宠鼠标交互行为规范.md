# 桌宠鼠标交互行为规范（BDD）

## 架构说明

桌宠是一个透明置顶的 Electron 窗口。窗口拖拽通过 CSS `-webkit-app-region: drag` 实现。

**关键约束**：drag 区域在 Electron 中会阻断 `click`、`dblclick` 等原生 DOM 事件。因此所有交互裁决必须基于 `mousedown` + `mouseup` 的坐标和时间差值来判断用户意图。

## 交互模型

```
mousedown
  ├─ 500ms 内无后续操作              → hover 开始（放大 1.06x）
  ├─ mouseup 移动 < 4px
  │   ├─ 350ms 内第二次 click         → 双击 → 切换聊天面板
  │   └─ 超时无第二次 click            → 单击 → 切换表情
  └─ mouseup 移动 >= 4px             → 拖拽 → 窗口跟随鼠标
```

---

## Feature 1：单击换表情

**As a** 用户
**I want to** 单击桌宠
**So that** 桌宠切换到下一个表情状态

### Scenario 1.1：快速单击
**Given** 桌宠处于某个表情状态
**When** 用户按下并抬起鼠标，移动距离 < 4px，350ms 内没有第二次点击
**Then** 桌宠切换到下一个表情  (idle → working → success → sleeping → idle ...)

### Scenario 1.2：点击时移动过大
**Given** 桌宠在桌面上
**When** 用户按下后移动 >= 4px 再抬起
**Then** 判定为拖拽，不触发表情切换

---

## Feature 2：双击切换聊天面板

**As a** 用户
**I want to** 双击桌宠
**So that** 打开或关闭聊天面板

### Scenario 2.1：350ms 内快速双击
**Given** 桌宠在桌面上
**When** 用户在 350ms 内快速点击两次（每次移动 < 4px）
**Then** 第一次单击被抑制（不换表情）
**And** 聊天面板打开（如果当前关闭）或关闭（如果当前打开且是聊天面板）

### Scenario 2.2：两次点击间隔过长
**Given** 桌宠在桌面上
**When** 两次点击间隔 > 350ms
**Then** 每次都是独立单击，各切换一次表情

---

## Feature 3：拖拽移动窗口

**As a** 用户
**I want to** 拖拽桌宠
**So that** 移动窗口到桌面任意位置

### Scenario 3.1：按住拖动
**Given** 桌宠在桌面上
**When** 用户按下桌宠并拖动 >= 4px
**Then** 窗口跟随鼠标移动
**And** 不触发任何点击事件

### Scenario 3.2：拖动结束
**Given** 用户正在拖拽桌宠
**When** 用户抬起鼠标
**Then** 桌宠停留在新位置
**And** 不触发单击或双击

---

## Feature 4：右键菜单

**As a** 用户
**I want to** 右键桌宠
**So that** 弹出功能菜单

### Scenario 4.1：右键点击
**Given** 桌宠在桌面上
**When** 用户右键桌宠
**Then** 弹出 Electron 原生右键菜单
**And** 菜单包含：聊天 / 待办&提醒 / Agent监听 / 记忆 / 生成分享图 / 待机 / 工作中 / 完成 / 勿扰 / 设置 / 收起面板 / 隐藏 / 退出

**实现说明**：Drag 区域阻断 Electron 的 webContents `context-menu` 事件，因此 renderer 的 `onContextMenu` handler 调用 `event.preventDefault()` 并通过 IPC 通知 main process 弹出原生菜单。

---

## Feature 5：悬停

**As a** 用户
**I want to** 鼠标停在桌宠上方时获得反馈
**So that** 桌宠看起来"注意到我"

### Scenario 5.1：静止悬停
**Given** 桌宠在桌面上
**When** 鼠标在桌宠上静止 500ms
**Then** 桌宠放大至 1.06 倍（CSS transform scale）

### Scenario 5.2：悬停结束
**Given** 鼠标正在桌宠上悬停
**When** 鼠标移开或点击
**Then** 桌宠恢复原始大小

---

## Feature 6：交互优先级

**As a** 系统
**I want to** 按优先级裁决鼠标行为
**So that** 一个鼠标动作不会触发多个冲突行为

### 优先级顺序
```
拖拽 > 双击 > 单击 > 悬停
```

### Scenario 6.1：拖拽取消一切
**Given** 用户已按下鼠标并开始悬停
**When** 用户拖动超过 4px
**Then** 悬停被清除，不触发任何点击事件

### Scenario 6.2：双击抑制单击
**Given** 用户第一次点击
**When** 用户在 350ms 窗口内第二次点击
**Then** 第一次的单击回调被取消，触发双击

---

## 技术实现要点

| 要点 | 说明 |
|------|------|
| 事件层 | 使用 React `onMouseDown` + `onMouseUp`（非原生 click/dblclick），因为 drag 区域会阻断后者 |
| 裁决器 | 纯函数 `InteractionDecider`，无 DOM 依赖，可独立测试 |
| 拖拽 | 由 CSS `-webkit-app-region: drag` 原生驱动（Electron 窗口管理器），裁决器只负责判定"这是拖拽" |
| 右键 | DOM `onContextMenu` → `event.preventDefault()` → IPC → main process `popup()` |
| 悬停 | `setTimeout(500ms)`，mousedown 时启动，mouseup/mouseleave 时清除 |
