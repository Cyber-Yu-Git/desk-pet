# 安全与隐私设计

## 1. 目标

本项目会接触聊天内容、待办、提醒、Agent 状态、项目路径和分享内容，因此必须从 MVP 开始建立安全边界。

目标：

- 本地优先。
- 明确权限。
- 不默认上传用户数据。
- API Key 不暴露给 renderer。
- 分享前必须预览和脱敏。
- 插件权限从第一版开始有记录。

## 2. Electron 安全基线

BrowserWindow 必须使用：

```ts
webPreferences: {
  contextIsolation: true,
  nodeIntegration: false,
  sandbox: true,
  preload: path.join(__dirname, '../preload/index.js')
}
```

禁止：

- 暴露完整 `ipcRenderer`。
- renderer 直接访问 Node.js。
- renderer 直接访问文件系统。
- 加载远程页面作为主 UI。
- 动态执行远程脚本。

## 3. Preload 白名单

Preload 只暴露：

- chat。
- todo。
- reminder。
- memory。
- agent。
- share。
- settings。
- events。

所有 API 必须是明确命名，不使用无限制通用方法。

不推荐：

```text
settings.get(anyKey)
settings.set(anyKey, anyValue)
```

推荐：

```text
settings.getPetConfig()
settings.updatePetConfig(input)
settings.getNotificationConfig()
settings.updateNotificationConfig(input)
```

## 4. IPC 校验

所有 `ipcMain.handle` 必须做运行时校验。

必须校验：

- 类型。
- 字段长度。
- 枚举值。
- 时间格式。
- 路径是否合法。
- 是否包含敏感字段。

TypeScript 类型不能替代运行时校验。

## 5. DeepSeek API Key

原则：

- API Key 只在 Main Process 使用。
- Renderer 不持有 API Key。
- API Key 不写入日志。
- API Key 不进入分享图。
- API Key 不进入错误提示。

存储策略：

1. 优先使用 Electron `safeStorage` 或系统安全存储。
2. 如果 MVP 暂时降级为本地存储，必须加密或至少明确标记为临时方案。
3. electron-store 只保存非敏感配置。

## 6. Content Security Policy

生产环境 CSP 建议：

```html
<meta
  http-equiv="Content-Security-Policy"
  content="default-src 'self'; script-src 'self'; style-src 'self' 'unsafe-inline'; img-src 'self' data:; connect-src 'self';"
/>
```

说明：

- Renderer 不直接请求 DeepSeek，因此不需要把 `https://api.deepseek.com` 放进 renderer CSP。
- DeepSeek 请求由 Main Process 发出。
- 开发环境可为 Vite dev server 放宽，生产必须收紧。

## 7. 窗口与导航保护

必须配置：

- 阻止任意新窗口。
- 阻止未知 URL 导航。
- 外部链接用系统浏览器打开。
- 权限请求默认拒绝。
- 仅白名单允许通知权限。

## 8. 本地数据

MVP 保存：

- 设置。
- 待办。
- 提醒。
- 少量聊天记录。
- 少量记忆。
- Agent 事件。
- 分享记录 metadata。

限制：

- 聊天记录默认最多保留最近 200 条。
- Agent events 默认最多保留最近 1000 条或 30 天，以先到者为准。
- 分享记录默认最多保留最近 50 条 metadata。
- 分享记录不保存大图 base64。
- P0 记忆最多保留 20 条。
- 用户可清空聊天、待办、记忆、Agent 日志。

## 9. 分享脱敏

分享前必须脱敏：

- API Key。
- Token。
- 邮箱。
- 手机号。
- 用户名。
- 绝对路径。
- 项目敏感文件名。
- 客户名或私密项目名。

分享流程：

```text
生成内容
  -> 脱敏
  -> 预览
  -> 用户确认
  -> 保存 PNG
```

禁止：

- 自动上传。
- 自动保存。
- 自动分享。
- 不预览直接导出。

## 10. 插件权限

插件 Manifest 必须声明权限。

敏感权限：

- `process:read`
- `window:read`
- `filesystem:watch`
- `notification:send`
- `memory:suggest`
- `network:deepseek`
- `storage:read`
- `storage:write`

MVP：

- 只做内置插件。
- 权限用于文档化和设置页展示。

P1：

- Plugin Runtime 调用系统能力前强制检查权限。

## 11. 日志策略

允许记录：

- 事件类型。
- 状态变化。
- 性能指标。
- 错误码。
- 脱敏后的路径或项目名。

禁止记录：

- API Key。
- Token。
- 完整聊天全文。
- 完整待办隐私内容。
- 完整项目路径。
- 用户文件内容。

## 12. 安全验收

MVP 必须满足：

- `nodeIntegration` 关闭。
- `contextIsolation` 开启。
- `sandbox` 开启。
- preload 白名单 API。
- DeepSeek API Key 不进入 renderer。
- CSP 生产环境收紧。
- IPC 输入校验。
- 分享前脱敏和预览。
- 本地数据可清空。
