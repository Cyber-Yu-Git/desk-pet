# MVP 内测验收清单

本文档用于内测前手动验收。每次发包前至少完整跑一遍 P0 检查。

## P0 必测链路

### 1. 启动与首次引导

- 启动应用后能看到桌宠窗口。
- 桌宠窗口透明、置顶、可拖动。
- 未配置 DeepSeek Key 时，默认进入“设置”面板。
- 设置面板显示 Key 配置状态、本机加密状态和数据目录。
- 点击“跳过”后首次引导不再反复出现。

通过标准：启动 10 秒内无崩溃，界面元素不重叠，首次用户知道下一步要配置 Key。

### 2. DeepSeek 聊天

- 在“设置”面板保存 DeepSeek API Key。
- 切到“聊天”面板，发送一条普通消息。
- 桌宠进入工作中状态，回复后进入完成状态。
- 关闭重启后，最近聊天记录仍能显示。
- 清空聊天后，聊天面板回到空状态。

通过标准：API Key 不出现在前端明文展示中；无 Key 时给出可理解错误；有 Key 时能正常回复。

### 3. 待办提醒

- 创建一条待办。
- 创建 1 分钟后提醒的待办。
- 待办列表显示标题和提醒时间。
- 点击完成后状态变为已完成。
- 点击删除后从列表消失。
- 到点提醒时桌宠进入提醒状态，并尝试发系统通知。

通过标准：新增待办不会立刻触发提醒状态；只有到点提醒才高优先级提醒。

### 4. Agent 状态模拟

- 创建一个 Trae 任务。
- 手动切换为运行中。
- 手动切换为等待确认。
- 手动切换为完成。
- 手动切换为失败。
- 删除任务。

通过标准：Agent 状态能驱动桌宠状态：运行中为工作中，等待确认为等待，完成为完成，失败为出错。

### 5. 分享图

- 在有聊天、待办或 Agent 任务后生成分享图。
- 分享图有预览。
- PNG 保存到 Windows 图片目录下的“赛博宇桌宠分享图”。
- 分享图中路径和疑似 Key 被脱敏。

通过标准：生成过程无崩溃，保存路径可访问，图片能正常打开。

### 6. 长期记忆

- 添加一条偏好记忆。
- 添加标签。
- 切换到聊天，发送一条和记忆相关的问题。
- 删除记忆。
- 清空记忆。

通过标准：记忆可增删查；聊天会读取高置信记忆；清空后列表为空。

### 7. 设置与数据管理

- 保存 DeepSeek Key。
- 清空 DeepSeek Key。
- 修改模型名。
- 打开数据目录。
- 清空聊天。
- 清空记忆。

通过标准：Key 使用本机加密保存；前端只显示是否已配置；清空操作立即反映到界面。

## P1 可选检查

- 安装包能正常安装。
- 免安装版能直接启动。
- 托盘菜单可用。
- 右键菜单可用。
- 退出后没有残留进程。


## 已知限制

- Agent Watcher 当前是模拟器，尚未真实监听 Trae / OpenClaw。
- 分享图模板仍是基础版。
- 长期记忆当前为本地 JSON 和简单 prompt 注入，尚未接向量检索。
- 当前 Windows 打包默认关闭 exe 元数据编辑，以避开部分环境的符号链接权限问题。
