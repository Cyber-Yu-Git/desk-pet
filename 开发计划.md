# 开发计划

## 1. 开发原则

本项目使用小步快跑方式开发。每个阶段都必须保持应用可运行，避免一次性生成大量不可维护代码。

原则：

- 每次只做一个可验收目标。
- 每个阶段结束都能录屏演示。
- 先模拟，后真实集成。
- 先 P0，后 P1。
- 先稳定体验，后扩展生态。
- 不在 MVP 阶段追求技术完美。

## 2. 阶段 0：项目初始化

目标：搭好可运行工程。

任务：

- 初始化 Electron + React + TypeScript + Vite + electron-vite。
- 配置 ESLint。
- 配置 `tsc --noEmit`。
- 配置 main / preload / renderer 三入口。
- 配置基础 BrowserWindow 安全选项。
- 创建空白桌宠窗口。

验收：

- `npm run dev` 可启动。
- `npm run typecheck` 可运行。
- 窗口打开后无控制台安全警告。
- renderer 无 Node.js 权限。

## 3. 阶段 1：桌宠窗口与动画

目标：桌宠可见、可拖拽、可动。

任务：

- 创建透明、无边框、置顶 Pet Window。
- 显示默认桌宠图片。
- 实现拖拽移动。
- 实现右键菜单。
- 实现托盘退出。
- 实现 Canvas sprite animator。
- 实现 `idle`、`working`、`success`、`error` 状态。

验收：

- 桌宠能常驻桌面。
- 拖拽不卡顿。
- idle 动画可持续播放。
- 状态切换不触发 React 逐帧重渲染。
- 初步性能面板可查看 FPS。

## 4. 阶段 2：Preload、IPC 和本地存储

目标：建立安全通信和本地数据基础。

任务：

- 实现 preload 白名单 API。
- 实现 `shared/ipcChannels.ts`。
- 实现 `shared/eventNames.ts`。
- 实现 `shared/types.ts`。
- 实现 IPC 参数校验。
- 接入 electron-store。
- Main Process 统一读写本地数据。

验收：

- renderer 不能直接访问 Node.js。
- renderer 不能直接访问 electron-store。
- 所有 IPC 通道有集中定义。
- 无约束 `settings:get/set(anyKey)` 不进入 P0。

## 5. 阶段 3：DeepSeek 聊天

目标：桌宠能进行文字聊天。

任务：

- 实现 DeepSeek API Key 配置。
- 实现 secretService。
- DeepSeek 请求放 Main Process。
- 实现聊天气泡。
- 实现请求超时和失败降级。
- 聊天记录本地保存最近 200 条。

验收：

- DeepSeek 在线可聊天。
- DeepSeek 离线不影响桌宠和待办。
- API Key 不出现在 renderer 和日志里。
- 聊天气泡可关闭、可复制回复。

## 6. 阶段 4：待办与提醒

目标：形成日常留存价值。

任务：

- 实现 Todo 数据模型。
- 实现 Reminder 数据模型。
- 实现创建、完成、删除待办。
- 实现提醒调度。
- 到点触发桌宠气泡。
- 到点触发系统通知。
- 支持稍后提醒、完成、忽略。

验收：

- 应用隐藏后提醒仍能触发。
- 启动后能恢复未完成待办。
- 过期提醒不会连续刷屏。
- DeepSeek 离线仍能手动创建待办。

## 7. 阶段 5：Agent 状态模拟器

目标：先跑通 Agent 提醒体验，不被真实工具集成拖住。

任务：

- 实现 AgentEvent 类型。
- 实现 Agent Simulator 面板。
- 手动触发 started、running、waiting_permission、completed、failed。
- Pet Engine 消费 AgentEvent。
- NotificationService 消费高优先级事件。
- completed 触发分享入口。

验收：

- Agent 状态变化 500ms 内反映到桌宠。
- waiting_permission 插队提醒。
- completed 触发 success 动画。
- failed 触发 error 动画。
- 调试日志能看到事件流。

## 8. 阶段 6：分享图

目标：形成传播闭环。

任务：

- 实现 SharePreview。
- 实现分享图 Canvas 或 HTML 渲染。
- 实现基础脱敏。
- 实现保存 PNG。
- 实现复制文案。
- Agent completed 后出现分享入口。

验收：

- 分享图可读。
- 默认不泄露敏感路径。
- 保存前必须预览。
- 不自动上传。
- 不做 GIF/MP4。

## 9. 阶段 7：P0.5 真实 Agent Watcher

目标：打通第一个真实 Agent 工具，但不阻塞 P0 发布。

任务：

- 选择 Claude Code / Codex / Trae / OpenClaw 中一个。
- 验证可监听来源。
- 实现 Adapter。
- 实现 Event Normalizer。
- 转换为标准 AgentEvent。
- 保留手动兜底入口。
- 如果真实集成卡住，跳过本阶段，先进入 P0 打磨。

验收：

- 能识别至少 3 类状态。
- 不依赖屏幕 OCR。
- 不高频扫描。
- 失败时不影响主流程。

## 10. 阶段 8：P0 打磨与内部测试

目标：准备小范围演示。

任务：

- 跑通 smoke test。
- 检查安全配置。
- 检查性能面板。
- 检查分享脱敏。
- 检查托盘退出。
- 整理演示脚本。
- 打 Windows 测试包。

验收：

- 连续运行 2 小时不崩溃。
- P0 演示路径完整，不依赖真实 Agent Watcher。
- 打包后可安装和卸载。
- 没有明显隐私泄漏。
- 至少 3 个测试用户能看懂价值。

## 11. P1 计划

P1 目标：增强留存和可配置性。

候选任务：

- 基础记忆系统。
- 记忆管理页。
- SQLite 迁移。
- 今日总结卡片。
- 分享模板。
- 真实 Agent Watcher 稳定化和第二个工具。
- 插件启用/禁用。
- 更精细鼠标穿透。

## 12. 每阶段固定检查

每阶段结束必须检查：

- 应用能启动。
- `typecheck` 通过。
- smoke test 通过。
- 无明显安全配置回退。
- 无明显全局状态膨胀。
- 无高频轮询。
- 文档是否需要同步更新。

## 13. 建议开发节奏

如果单人 + AI 辅助开发：

```text
第 1 周：阶段 0-1
第 2 周：阶段 2-3
第 3 周：阶段 4
第 4 周：阶段 5-6
第 5 周：阶段 7，若真实 Agent 集成卡住则跳过
第 6 周：阶段 8
```

如果遇到真实 Agent 集成卡住，不阻塞 P0，先保留模拟器和手动兜底。
